
# **“Persona” AI 社交应用开发任务清单**

> **项目名称:** **“Persona” —— 构建你的 AI 化身，定义下一代社交网络**
> 最终交付日期: **2025年11月30日**

## 核心开发理念与注意事项

* **迭代开发，循序渐进：** **强烈建议**完整地**完成一个阶段（如：基础要求）的所有核心功能后，再开始下一个阶段（进阶挑战）。这能确保你始终有一个可运行、可演示的版本。**
* **版本控制是生命线：** **从项目初始化就使用 Git。为每个功能或模块创建独立的** **feature** **分支，完成开发和测试后再合并到主分支。保持提交信息（Commit Message）的清晰规范。**
* **文档先行，记录思考：** **开发过程中的每一个重要决策（为什么选这个库？为什么这样设计架构？）都应随手记录。这不仅有助于你梳理思路，更是最终深度报告的宝贵素材。**
* **拥抱AI，但保持清醒：** **充分利用 AI 编程助手提升效率，但必须确保你**完全理解 **AI 生成的每一行代码。在最终报告中，你需要清晰说明在哪些环节、以何种方式使用了 AI 工具。**
* **用户体验贯穿始终：** **即使是早期版本，也要注重 UI 的流畅性和交互的直观性。一个体验良好的应用更能打动人心。**

## 第一阶段：基础要求 —— 跑通核心 MVP (Minimum Viable Product)

**此阶段的目标是构建一个功能完整、可独立演示的核心应用，证明你的基础工程落地能力。**

* **1. 项目初始化与环境搭建**

  * **创建 Git 仓库，并规划好分支策略 (e.g.,** **main**, **develop**, **feature/...**)。
  * **确定客户端技术栈 (e.g., Swift/SwiftUI, Kotlin/Jetpack Compose, Flutter, React Native)。**
  * **搭建基础项目结构，规划好模块/目录（如：**Models**,** **Views**, **ViewModels**, **Services**）。
  * **引入基础依赖库（网络请求、UI组件等）。**
* **2. Persona 创作模块**

  * **UI/UX 实现：**

    * **创建引导用户自定义 Persona 的界面（输入名称、头像、性格标签、背景故事等）。**
  * **功能逻辑：**

    * **实现 Persona 数据模型（Model）。**
    * **[AI 融合]** **集成一个云端大语言模型 API（如 OpenAI, Google Gemini, Anthropic Claude 等）。**
    * **实现 “AI 辅助生成” 功能：调用 API，根据用户输入的少量关键词，一键生成完整的 Persona 设定。**
    * **使用 Mock 数据/本地临时存储来保存创建的 Persona。**
* **3. Persona 社交模块**

  * **UI/UX 实现：**

    * **社交广场：** **实现一个信息流（Feed）界面，用于展示所有 Persona 的动态（类似微博/Twitter）。**
    * **动态详情：** **点击动态可进入详情页。**
    * **Persona 主页：** **点击 Persona 头像可进入其主页，展示其设定和历史动态。**
  * **功能逻辑：**

    * **实现动态（Post）数据模型。**
    * **[AI 融合]** **在 Persona 主页或特定界面，提供一个“触发生成动态”的按钮，调用 LLM API 根据其人设生成图文动态内容。**
    * **实现关注/取关功能。**
    * **数据来源：** **此阶段所有广场动态、关注关系均使用**本地 Mock 数据**。**
* **4. 直接对话模块**

  * **UI/UX 实现：**

    * **实现一个标准的聊天界面（对话气泡、输入框、发送按钮）。**
  * **功能逻辑：**

    * **从任意 Persona 主页发起聊天，进入对话界面。**
    * **[AI 融合]** **对接云端 LLM API，将用户的输入和 Persona 的人设（作为 System Prompt）一同发送，获取并展示 AI 的回复。**
    * **注意：** **按要求，此阶段聊天记录**无需本地存储**。**
* **5. Persona 共生（私聊）模块**

  * **UI/UX 实现：**

    * **创建一个独立的、用户与自己 Persona 的专属私聊界面。**
  * **功能逻辑：**

    * **实现与自己 Persona 的对话功能（逻辑同“直接对话”模块）。**
    * **核心区别：** **思考如何通过对话来“调整”Persona 的设定（此阶段可只做概念，或用简单的机制实现，如记录下关键对话作为未来 Prompt 的一部分）。**

## 第二阶段：进階挑戰 —— 提升品质与深度

**完成基础要求后，从以下任务中选择挑战，以展现你的技术深度和创新能力。建议按 P0 -> P3 的顺序逐步攻克。**

### [P0] 功能体验优化

* **1. 富文本渲染**

  * **调研并集成一个支持 Markdown 渲染的客户端库。**
  * **确保 Persona 生成的文本（如动态、聊天回复）能正确解析并展示 Markdown 格式（加粗、列表、引用等）。**
* **2. 流式输出**

  * **改造聊天模块的网络请求和 UI 更新逻辑。**
  * **实现大语言模型回复的“打字机”式流式展示效果，提升交互的即时感。**

### [P1] 多模态交互

* **1. [AI 融合] 多模态模型集成**

  * **调研并选择一个文生图（如 Stable Diffusion, Midjourney API）、文生音乐或 AI 语音（TTS）的云端 API。**
  * **在项目中集成该 API，让 Persona 能够根据指令生成图片、音乐或发送语音消息。**
  * **设计相应的 UI 来承载和展示这些多模态内容。**

### [P2] 项目架构优化

* **1. 多账户体系与数据隔离**

  * **设计并实现用户模型（User Model）。**
  * **创建登录/注册/账户切换的 UI 和业务逻辑。**
  * **重构数据管理层，确保所有数据（Personas, Posts, Follows）都与特定用户账户关联，实现数据隔离。**
* **2. “可插拔”数据源架构**

  * **设计抽象层：** **定义清晰的数据仓储（Repository）或服务（Service）接口，将数据来源（本地 Mock vs. 真实后端）与业务逻辑解耦。**
  * **实现不同数据源：**

    * **保留现有的 Mock 数据实现。**
    * **创建一套新的、模拟调用真实后端 API 的实现（可以只是定义接口和数据结构，无需真实后端）。**
  * **验证可切换性：** **证明可以仅通过修改少量配置代码，就能在两种数据源之间无缝切换。**

### [P3] 前沿探索与深度复盘

* **1. [AI 融合] 端云协同混合 AI 架构**

  * **端侧模型调研：** **研究当前主流的轻量级端侧大模型（如 Llama 3-8B, Phi-3-mini, Gemma 2B 等）及其在移动端的部署方案（如 llama.cpp, MediaPipe, CoreML, TFLite）。**
  * **端侧模型部署：** **选择一个模型，完成量化、转换，并成功集成到 App 中。**
  * **功能实现：** **使用部署好的端侧模型，驱动“Persona 共生（私聊）”模块，实现完全离线对话。**
  * **混合架构设计：** **设计并实现一个策略控制器（Policy Controller），用于智能决策何时调用端侧模型（快、隐私、免费），何时调用云端模型（强、多模态、创造力）。**
* **2. [算法思考] 智能推荐系统**

  * **理论设计：** **在报告中详细阐述“Persona 发现”推荐算法的原理。你可以选择协同过滤、基于内容的推荐，或是更简单的基于标签/热度的推荐算法。**
  * **系统阐述：** **绘制数据流图和系统架构图，说明需要哪些数据、数据如何处理、推荐结果如何生成。**
  * **(可选) 简单实现：** **在 App 的“发现”页中，基于 Mock 数据实现一个简化版的推荐逻辑。**
